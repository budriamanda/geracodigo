### O Ângulo Correto: "gerador de código" como Hub

`"gerador de código"` (sem "de barras") é um termo **mais amplo e ambíguo**. Quem busca isso pode querer:
- Gerador de código de barras
- Gerador de QR Code
- Gerador de QR Code Pix
- Gerador de SKU
- Gerador de código de programação (fora do escopo)

Isso é intent **informacional + navegacional** — o usuário ainda está descobrindo qual ferramenta precisa. Nenhuma das tool pages cobre isso. É o ângulo ideal para um artigo que **serve de hub** para todo o ecossistema GeraCode.

**Gap identificado:** Nenhum concorrente cobre o ecossistema completo (código de barras + QR + Pix + SKU) com foco em lojistas brasileiros em um único artigo orientado à decisão.

### O que os concorrentes NÃO fazem
- Não explicam a diferença entre os tipos de código para quem está começando
- Não cruzam necessidades de negócio com o tipo certo de código
- Não mostram que uma mesma loja pode precisar de 2-3 tipos simultaneamente

## 3. Esboço Recomendado

**Conceito:** Artigo de descoberta/decisão que responde "qual gerador de código eu preciso?" e direciona para as ferramentas certas.

```
H1: Gerador de Código Grátis Online: Qual Usar para Cada Caso?

Introdução (100 palavras)
- Hook: lojistas usam vários tipos de código sem saber a diferença
- Proposta: em 3 minutos, você saberá exatamente qual usar
- Sem cadastro, todos grátis

H2: Os 5 tipos de código que lojistas brasileiros usam
  H3: Código de barras (EAN-13, Code 128...) → link /gerador-de-codigo-de-barras
  H3: QR Code → link /gerador-de-qr-code
  H3: QR Code Pix → link /gerador-de-qr-code-pix
  H3: EAN-13 específico → link /gerador-de-ean
  H3: SKU → link /gerador-de-sku

H2: Tabela de decisão — qual código para qual necessidade?
  (tabela: necessidade x código recomendado x onde gerar)
  - "Quero vender em supermercado" → EAN-13
  - "Quero controlar meu estoque" → Code 128 ou SKU
  - "Quero receber pagamento via Pix" → QR Code Pix
  - "Quero cardápio digital" → QR Code
  - "Etiquetar caixas de transporte" → ITF-14
  - "Vender nos EUA" → UPC-A

H2: Posso usar mais de um tipo ao mesmo tempo?
  - Exemplo: restaurante usa QR Code (cardápio) + QR Pix (pagamento) + Code 128 (estoque)
  - Exemplo: loja física usa EAN-13 (produto) + Code 128 (estoque) + SKU (interno)

H2: Por que gerar online e grátis é suficiente (quando não preciso da GS1)
  - Casos onde o código grátis basta
  - Casos onde a GS1 é obrigatória → link /blog/codigo-de-barras-para-produtos-guia-completo

H2: Perguntas frequentes
  - Todos os geradores são grátis?
  - Meus dados ficam seguros?
  - Funciona em celular?

Conclusão + CTA para cada ferramenta
```
